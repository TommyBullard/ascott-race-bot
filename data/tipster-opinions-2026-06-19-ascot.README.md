# Tipster opinions — 2026-06-19 Ascot (compliant ingestion workflow)

A source-compliant way to turn **real, permitted** tipster opinions into
model-active `tipster_selections` — only after evidence + licence + review checks.
The example files here are **EXAMPLE/template** content; replace them with real,
licence-cleared notes. Nothing in the templates can become model-active.

## Source policy

**Allowed**
1. Operator manual notes / first-party observations (`licence_status: manual`)
2. Public pages explicitly allowed for reuse (`public_allowed`)
3. Licensed API / feed notes (`licensed`)
4. Short operator-supplied excerpts **with a source label**
5. Racing Post **only** as a licensed / short manual-excerpt source — **never scraped**

**Disallowed (never do this)**
1. Scraping paywalled Racing Post articles
2. Scraping logged-in / private tipster areas
3. Storing full copyrighted articles (use **short excerpts** only)
4. Letting OpenAI browse freely / guess selections without source evidence
5. Using AI-extracted opinions without evidence + human review

**Jon Vine / "What Would Jon Vine Do"**: only ingest real Jon Vine tips from a
**permitted** source, with evidence. Otherwise the profile is a **synthetic
strategy heuristic** — labelled, shadow-only until backtested, and **never**
treated as a real sourced tipster (it is forced to `licence_status = unknown` and
blocked from model-active use).

## Files

| File | Purpose |
| --- | --- |
| [tipster-notes-2026-06-19-ascot.json](tipster-notes-2026-06-19-ascot.json) | LOCAL notes input for extraction (structured; short excerpts + `source_text`) |
| [tipster-opinions-2026-06-19-ascot.csv](tipster-opinions-2026-06-19-ascot.csv) | The opinions CSV (extractor output / hand-edit) |
| `tipster-opinions-2026-06-19-ascot-approved.csv` | Generated by the review step — only approved + eligible rows, in importer format |

## CSV columns

`date, course, race_name, off_time, source_label, tipster_name, runner_name,
opinion_type, confidence, evidence_excerpt, source_url, licence_status, notes,
review_status`

- `opinion_type`: `selection | positive | negative | each_way_interest | danger | no_strong_view` — **only `selection`** can become a backing `tipster_selection`.
- `confidence`: `low | medium | high | unknown`
- `licence_status`: `manual | public_allowed | licensed | unknown` — **`unknown` can never be model-active**.
- `review_status`: `pending | approved | rejected` — extraction always emits `pending`.
- `evidence_excerpt`: **required**, a short verbatim snippet; for extraction it must appear in the note's `source_text` (anti-fabrication grounding).

## Workflow (safe; dry-run first)

```bash
# 1. EXTRACT from LOCAL notes only (no scraping, no OpenAI guessing; evidence-grounded):
npm run tipsters:extract-opinions -- --input data/tipster-notes-2026-06-19-ascot.json \
  --output data/tipster-opinions-2026-06-19-ascot.csv

# 2. REVIEW against today's real runners (read-only); writes *-approved.csv with ONLY
#    rows that are review_status=approved AND eligible (matched + permitted licence +
#    evidence + a real selection + a permitted, non-synthetic source):
npm run tipsters:review-opinions -- --file data/tipster-opinions-2026-06-19-ascot.csv

# 3. Set review_status=approved on the rows you have vetted (real, permitted picks only),
#    re-run step 2, then IMPORT the approved selections (WRITE — operator approval):
npm run import:tipster-selections -- --file data/tipster-opinions-2026-06-19-ascot-approved.csv --commit

# 4. VERIFY consensus (read-only):
npm run verify:tipster-match -- --date 2026-06-19 --course Ascot

# 5. Re-run the model so matched selections become model-active:
npm run pipeline:day -- --date 2026-06-19 --course Ascot --commit
```

A matched, approved selection clears `NO_TIPSTER_CONSENSUS` **only** for the races
it affects. Stale historical selections (e.g. 2026-06-16) are **not** counted as
support for today's races. This workflow changes **no** model/EV/staking/
recommendation math and adds **no** auto-betting or bet placement.
